Environment Variables

Kubernetes environment variables by example

You can set environment variables for containers running in a pod and in addition, Kubernetes exposes certain runtime infos via environment variables automatically.

Let’s launch a pod that we pass an environment variable SIMPLE_SERVICE_VERSION with the value 1.0:

kubectl apply -f https://raw.githubusercontent.com/openshift-evangelists/kbe/main/specs/envs/pod.yaml

Now, let’s verify from within the cluster if the application running in the pod has picked up the environment variable SIMPLE_SERVICE_VERSION:

kubectl exec envs -t -- curl -s 127.0.0.1:9876/info

{"host": "127.0.0.1:9876", "version": "1.0", "from": "127.0.0.1"}

And indeed it has picked up the user-provided environment variable since the default response would be "version": "0.5.0".

Alternatively, you can also use kubectl exec to display the environment variables within the container:

kubectl exec envs -- printenv

PATH=/usr/local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
HOSTNAME=envs
SIMPLE_SERVICE_VERSION=1.0
KUBERNETES_PORT_53_UDP_ADDR=172.30.0.1

KUBERNETES_SERVICE_PORT_DNS_TCP=53
ROUTER_PORT=tcp://172.30.246.127:80
...

Remove the env pod with:

kubectl delete pod/envs

In addition to the above examples, you can also use secrets, volumes, or the downward API to inject additional information into your container environments.
